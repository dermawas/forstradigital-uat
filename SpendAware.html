SpendAware — App Brief

Feasibility · PRD Skeleton · Tech Stack  |  Forstra Digital · June 2026

Feasibility verdict

✅ Fully buildable as a vibe-coding project. MVP 1 (manual tracking + threshold alerts) is achievable in 2–4 weeks with Expo + Supabase. MVP 2 OCR feature is the right call over Open Banking — cheaper, works across all banks, no API approval needed.

Feature scope by phase

MVP 1 Core tracking
  • Manual expense entry — amount, category, note, date
  • Preset categories (makan, transport, tagihan, hiburan, dll) + custom
  • Monthly balance view: income – expenses = remaining
  • Budget per category — set limit, track remaining
  • Spending threshold alert — push notif at 80% and 100% of budget
  • Monthly summary screen with simple bar/pie breakdown
MVP 2 OCR receipt capture
  • User taps "Scan" → takes photo of bank notif screenshot or paper receipt
  • App extracts: merchant name, amount, date
  • Pre-fills entry form — user confirms + picks category
  • Works on ANY bank — no API dependency
  • Tech: Google ML Kit Text Recognition (on-device, free, offline)

This IS your differentiator — 3-tap entry vs competitors' full manual. "Scan & confirm" is a strong UX story.

MVP 3 Freemium + growth
  • Free tier: 1 wallet, 3 category budgets, 30-day history
  • Premium: unlimited wallets, unlimited categories, full history, export CSV
  • Multi-wallet support (cash, BCA, GoPay, dll)
  • Recurring expense detection
  • Weekly digest notification

Tech stack

Mobile framework
React Native (Expo SDK 51+)
Android-first, iOS-ready later. One codebase. Expo Go = test on your phone in minutes, no store needed.
Backend + DB
Supabase (hosted, free tier)
PostgreSQL under the hood — familiar from GnuCash. Auth, realtime, REST API all included. Self-hostable on OCI later if needed. Free tier: 500MB DB, 50k MAU.
Auth
Supabase Auth (email + Google OAuth)
Built-in. Zero extra work.
OCR (MVP 2)
react-native-mlkit-ocr
Google ML Kit, on-device, free, works offline. No server call needed.
Push notifications
Expo Notifications (FCM for Android)
Threshold alerts at 80%/100% budget. Free via Firebase Cloud Messaging.
Charts
Victory Native or react-native-gifted-charts
Lightweight, no native module issues with Expo.
State management
Zustand (lightweight)
Simpler than Redux, works well with Supabase realtime.
Android distribution
EAS Build (Expo) → Google Play Console
EAS Build free tier = 30 builds/month. You already have a Play Console account.
Hosting (backend)
Supabase cloud (free to start) → OCI migrate later
No need for Ferdinand's OCI account at MVP 1. Revisit at MVP 3 if cost becomes a factor.
